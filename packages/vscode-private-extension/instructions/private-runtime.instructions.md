Use the local workspace as the implementation surface.

Treat the connected private backend or MCP service as the authoritative source for proprietary orchestration logic that is not bundled with the extension.

Do not claim full private-framework visibility from local extension assets alone.

Prefer the minimum required local context and request backend assistance when runtime-specific intelligence is required.

Preserve repository-native implementation behavior.
